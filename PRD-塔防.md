## 1. 项目概述

### 1.1 背景
塔防游戏（TD，Tower Defense）是网页与移动端最长寿的游戏品类之一（代表作：Bloons TD、Kingdom Rush）。它融合了策略、资源管理、波次设计、敌人 AI、塔的射程与攻击算法等多维度系统，是真正"完整游戏"的代表。

### 1.2 目标
- **学习目标**：掌握精灵系统、敌人 AI（A* 寻路）、波次管理、经济系统、范围攻击算法、动画状态机。
- **产品目标**：交付一款 5 关卡、3 种塔、5 种敌人的完整塔防游戏。

### 1.3 范围
**包含**：5 个内置关卡、3 种塔、5 种敌人、波次系统、金币经济。
**不包含**：多人联机、英雄系统、抽卡系统、内购。

## 3. 核心玩法

### 3.1 玩法描述
玩家在一张固定地图上建造防御塔。敌人（怪物）从起点出发沿固定路线前进，试图到达终点（基地）。塔自动攻击进入射程的敌人。当所有敌人被消灭或漏掉一定数量时关卡结束。

### 3.2 核心机制

| 机制 | 说明 |
| --- | --- |
| **金币（Gold）** | 击杀敌人获得金币，用于建造/升级塔 |
| **生命值（Lives）** | 敌人到达基地扣 1 生命，归零则失败 |
| **波次（Wave）** | 固定 N 波敌人，每波 N 只 |
| **建造** | 点击空地 → 选择塔 → 花费金币建造 |
| **升级** | 已建造的塔可花费金币升级（3 级） |
| **射程** | 塔的攻击范围，超出射程的敌人不可攻击 |

### 3.3 计分与胜负
- **胜利**：通过所有波次，基地生命 > 0。
- **失败**：基地生命归零。
- **得分**：通关时剩余生命 × 100 + 剩余金币。

### 6.2 视觉规范
- **地图**：16×12 网格，每格 40×40 px。
- **配色**：
  - 路径 `#8b7355`（土路色）
  - 草地 `#5a8c4a`（深绿）
  - 装饰 `#3a6b35`（石头/树）
  - 塔的范围圈 `#fbbf24` 半透明
- **塔图标**：
  - 基础弓手塔：蓝绿色，射箭
  - 减速塔：浅蓝色，射冰
  - 范围塔：红橙色，炮弹

### 6.3 动效
- 塔攻击：子弹从塔尖飞向敌人。
- 击中：敌人闪白一下，掉血数字向上飘。
- 敌人死亡：爆炸粒子。
- 塔升级：粒子 + 缩放反弹。
- 波次开始：屏幕顶部弹出 "Wave X"。

## 8. 数据结构关键设计

### 8.1 敌人（Enemy）
```ts
type EnemyType = 'normal' | 'fast' | 'heavy' | 'flying' | 'boss';

interface Enemy {
  id: string;
  type: EnemyType;
  hp: number;
  maxHp: number;
  speed: number;          // px/s
  reward: number;         // 死亡奖励金币
  pathIndex: number;      // 当前路径节点索引
  position: Point;
  path?: Point[];         // A* 路径
  status: 'alive' | 'dead' | 'escaped';
}
```

### 8.2 塔（Tower）
```ts
type TowerType = 'archer' | 'frost' | 'cannon';

interface Tower {
  id: string;
  type: TowerType;
  level: 1 | 2 | 3;
  position: Point;
  range: number;
  damage: number;
  attackSpeed: number;    // 次/秒
  lastAttackTime: number;
  target?: Enemy;
  cost: number;
}
```

### 8.3 波次（Wave）
```ts
interface Wave {
  index: number;
  enemies: { type: EnemyType; count: number; interval: number }[];
  reward: number;        // 通关奖励
}
```

### 8.4 子弹（Projectile）
```ts
interface Projectile {
  id: string;
  towerId: string;
  target: Enemy;
  position: Point;
  speed: number;
  damage: number;
  effect?: 'slow' | 'splash';  // V2 效果
}
```

## 9. 技术约束

### 9.1 引擎选型
- **方案 A（推荐）**：Phaser 3
  - 自带精灵系统、动画、物理、场景管理。
  - 学习曲线中等，社区活跃。
- **方案 B**：原生 Canvas 2D
  - 完全可控，零依赖，但开发周期长。
  - 适合想深入理解渲染管道的学习者。

### 9.2 状态机
```
MENU → LEVEL_SELECT → PLAYING → WAVE_INTERMISSION ↔ PLAYING
                            ↓
                         WIN / FAILED
```

### 9.3 系统模块
- `Engine` 引擎封装（Phaser Scene）
- `Entity` 实体基类（敌人、塔、子弹继承）
- `System` 系统模块（寻路、攻击、波次、经济）
- `UI` 用户界面（顶栏、塔选择面板、弹窗）
- `Data` 关卡 / 塔 / 敌人配置数据

### 9.4 性能要求
- 实体池（Object Pool）复用，避免频繁创建销毁。
- 空间分割（如均匀网格）优化大量敌人的射程检测。
- 离屏渲染优化子弹和粒子。

## 10. 风险与依赖

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 关卡难度设计失衡 | 高 | 反复测试 + 调节数值 |
| 同时 100+ 敌人卡顿 | 中 | 实体池 + 空间分割优化 |
| 塔升级路径数据膨胀 | 低 | 用 JSON 配置化 |
| 飞行敌人寻路与传统敌人不同 | 中 | 飞行敌人直线飞行，绕过塔 |
